Market Validation

In our fast-paced society, it’s easy for work, play, and health to jumble into a blur. Research has proven that the average human’s attention span is a mere 8 seconds -- that's shorter than that of a goldfish! It’s no wonder we often forget how many calories we have been eating or how often we forget how many times we have gone to the gym.

After digging a bit deeper into this inconvenience, we discovered that the time it takes someone to forget a memory due to the decay theory (a theory that states that memory fades as more time passes) has shortened to as little as 30 minutes. Without something to keep track of the little things in life like counting the hours one has spent meditating or the number of days one has gone without indulging in a guilty pleasure, people will continue to live without a method to stay organized and up to date on their own goals.

After running a survey in our community in Denton, Texas, a whopping 8/10 people said that they wanted a single app to keep track of all the counting they must memorize in a day. Thus, we wanted to create a counting app that promotes positive thinking and an organized lifestyle. From our color palette to the design of the interface, every aspect of Mindful Minutes has the purpose of creating the optimal environment for our users to have a fully functional lifestyle.

Upon further research, we discovered that adults forget at least 3 key facts, events, or chores every day. Thus, we knew that our app would have a universal market. We attempted to find counting apps online that would keep track of daily events in terms of units, time, and more. However, there were no apps that combine all of these components into one easy-to-use app. We knew that our app would create an impact on not only our local community, but it would revolutionize how people live their lives all around the world.

Moreover, we wanted to create an app based on positive feedback because it is scientifically proven that people respond with greater productivity, greater concentration, and greater motivation with positive feedback. Therefore, we wanted to allow the user to set goals for oneself so that once that goal is achieved the user is rewarded with a supporting congratulations method. It was also a goal of ours to keep the user motivated, so we wanted to create an option to restart and set a new goal when the user completes his/her prior goal.

Q&A Discussion

Q. What technical difficulties did you face programming your app?
When it came to programming the app, we initially struggled with the collaborative coding aspect of the app. Because nearly every feature of the app is dependent on another feature, we had to ensure we were both following the same coding conventions and named variables that made sense to both of us. We concluded that we would be using camel case to name our variables.

Furthermore, we had a document that noted all of the functions we created as well as a brief description of the function so that we knew which ones to call when needed. Initially, we had this information commented above each function. However, we found that this was inefficient due to the large number of features we had; it was an arduous process scrolling through each function in the code to identify what it did then scroll back to where we worked on our own section of the code. With a separate document, we could simply use “cmd+f”/"ctrl+f" to identify the function we needed to use.

The main technical difficulty we had was connected all the screens to where once we add a number on a counter’s add screen it would update on the actual counter screen. Our first attempt was to simply add the user input to the ID of the counter. However, because the ID was on a separate screen, this did not work. To solve this issue, we took the user input into the console and saved the number in the memory. Using this method, we were able to get the user input to accurately show up on a different screen.
